Rails

March 10, 2016

As we get a firmer grasp on the fundamentals of Ruby, it's only natural to ask the question: "What the heck is Rails?"

Rails Quick Facts

  • An extremely productive web-application framework.
  • Written in Ruby by David Heinemeier Hansson.
  • You could develop a web application at least ten times faster with Rails than you could with a typical Java framework.
  • An open source Ruby framework for developing database-backed web applications.
  • Configure your code with Database Schema.
  • No compilation phase required.

Full Stack Framework

Rails is not a programming langauge. Rather, it is a full stack framework, which means all of the layers work perfectly together with less code. It includes everything needed to create a database-driven web application, using the Model-View-Controller pattern. Model-View-Controller pattern consists of Ruby classes (Models) that deal with logic, Templates (Views) that take info from the models and show it to the user, and the Controller which exists between the models and views to handle requests.

Why Use Rails?

Rails has many features that make you more productive. A few of these include:

Metaprogramming:

Instead of relying on generating code from scratch, Rail framework uses Metaprogramming. "Metaprogramming is the writing of computer programs with the ability to treat programs as their data. It means that a program could be designed to read, generate, analyse or transform other programs, and even modify itself while running."

Active Record:

Rails uses Active Record to save objects into the database.

Conventions:

If you follow suggested naming conventions, Rails doesn't need much configuration.

The bottom line is: Rails is a framework written in Ruby that will make your life as a programmer easier. Sounds pretty good to me.
