Ruby Classes

February 26, 2016

Ruby is an object-oriented programming language and has both classes and objects. A class is a combination of characteristics and functions, and is the blueprint from which individual objects are created. For example, an individual vegetable is an instance of the class of objects known as vegetables. The various characteristics of any vegetable make up the data members of the class vegetables.

Let's take a look at creating a class in action. 

          
        class Vegetable
            def set_name(string)
              puts "Setting vegetable name..."
              @name = string
            end

            def get_name
              puts "Returning the vegetable name..."
              @name
            end
        end

        kale = Vegetable.new
        kale.set_name("kale")
        puts kale.get_name

Now let's break this down into smaller steps. To create the class we must use the word class plus the capitalized class name. 

        
          class Vegetable
          end

Then we define our method set_name that takes a string as a parameter. After the puts comes our first instance variable. Instance variables are prefixed with a single @ sign. Instance variables differ from local variables in that they are scoped to the class and can be accessed by any method in the class. This means that when we move along to our next method, we will still be able to access the @name instance variable even though it was created in the previous method. In our example, we set the instance variable @name = to the string input. The inputted string will now be accessible in any method in the class through the instance variable @name. 

        
          class Vegetable
            def set_name(string)
              puts "Setting vegetable name..."
              @name = string
            end

Next we create our second method get_name which takes 0 parameters. Checkout the instance variable from our previous method @name before the end of the method. This will carry the string input from the first method into this second method. 

            
            def get_name
              puts "Returning the vegetable name..."
              @name
            end

Finally, we use our driver code to create a new vegetable, kale, and call both the set_name method, as well as the get_name method. Both of the methods contained within our class are actually instance methods. Instance methods can only be called on a particular instance of the class. If we tried to call either of our instance methods directly on the class, we would get an error. Therefore, we must create a new instance of Vegetable with Vegetable.new so that we can call our instance methods. 

      
        kale = Vegetable.new
        kale.set_name("kale")
        puts kale.get_name
Our output will look like this: 
      
        Setting vegetable name...
        Returning the vegetable name...
        kale
