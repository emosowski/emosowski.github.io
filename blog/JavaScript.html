JavaScript Objects vs. Ruby Hashes

March 5, 2016

Starting to learn JavaScript after getting a basic grasp on Ruby can be overwhelming. Luckily, in many cases, once you become familiar with the different words and commands that correspond between the two languages, the syntax and behavior is actually pretty similar.

This is, in fact, the case when it comes to JavaScript objects and Ruby hashes. Objects can be used to contain a collection with values. Sound familiar? Yep, sounds a whole lot like a hash in Ruby. Now we get into the naming differences between the two languages. In JavaScript, objects have properties with a respective value. In Ruby, hashes have keys with a respective value. It is important to note that while in Ruby hash keys can be any type, properties can only be variables. Let's take a look first at an object in JavaScript and then at a hash in Ruby to see just how similar they look and behave. 

          
        var foodGroups = {
          meat: "chicken",
          vegetable: "kale",
          fruit: "apple",
          grain: "quinoa",
        };

        // To access a JavaScript value use dot notation with the property name
        foodGroups.meat;
        // => "chicken"


        // To change a JavaScript value use dot notation with
        // the property name and set it equal to the new value
        foodGroups.vegetable = "spinach";

        // To add a new property and value use dot notation with the new property name
        // and set it equal to the value
        foodGroups.dairy = "cheese";

        // To delete an existing property or value use delete and dot notation with the property
        // name. This will delete both property and value.
        delete foodGroups.fruit;

        // To write out the object to the text output device
        console.log(foodGroups);

        var foodGroups = {
          meat: "chicken",
          vegetable: "spinach",
          grain: "quinoa",
          dairy: "cheese",
        };

Now let's refresh our memories of how a hash works in Ruby. 

          

        food_groups = {
          meat: "chicken",
          vegetable: "kale",
          fruit: "apple",
          grain: "quinoa",
        }

        # To access a Ruby hash value use brackets and the key
        food_groups[:meat]
        # => "chicken"

        # To modify a value use brackets and the key equal to the new value
        food_groups[:vegetable] = "spinach"

        # To add a new key and value use bracket notation and the new key equal to the value
        food_groups[:dairy] = "cheese"

        # To delete an existing key and value use delete with the key
        food_groups.delete(:fruit)

        # To put the hash to the text output device
        puts food_groups

        food_groups = {
          meat: "chicken",
          vegetable: "spinach",
          grain: "quinoa",
          dairy: "cheese",
        }

Comparing the two, we can see that they are very similar in how they look and function. Differences in names and slight differences in syntax may feel cumbersome at first. Once we look at both an object in JavaScript and a hash in Ruby side by side, turns out it's not so difficult to switch between the langauges after all.
