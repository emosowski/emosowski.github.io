Enumerable Method: Enumerable#map

February 21, 2016

Enumerable#map is one of the most powerful methods in Ruby. map returns the results of calling the code block on each element in the original object. map always returns an array filled with these results that is equal in size to the original, regardless of what enumerable it starts the method with.

Let's look at map in action. 

          

            names = %w{ Emily Sarah Jenny Kate }
            => ["Emily, "Sarah", "Jenny", "Kate"]
            names.map {|name| name.upcase }
            => ["EMILY", "SARAH", "JENNY", "KATE"]

The most important thing to remember about map has to do with its return value. The return value of map is also what sets it apart from each. Every time map goes through the code block, the value returned by the block is accumulated into the result array. On the other hand, every time each goes through the code block,the value returned is discarded. Therefore, it is important to decide whether each, or map is more appropriate for what you are trying to do.

Because of the return of map, you have to be careful with block evaluation. For example: 

          

            array = [5,6,7,8,9]
            result = array.map {|n| puts n * 50 }
            => [nil, nil, nil, nil, nil]
As you can see, this use of map returned all nil. This is because the return value of puts is always nil. While the five values will get printed to the screen, the result of the mapping is all nils because the call to this block returns nil.
